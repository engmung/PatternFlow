# Patternflow GitHub 저장소 뼈대 구축

> 이 문서는 다른 AI 환경에서 실행할 작업 계획서입니다.
> 목적: Patternflow 프로젝트의 공개 GitHub 저장소 초기 구조 세팅.

---

## 프로젝트 컨텍스트

- **Patternflow**: 4개의 로터리 엔코더로 128×64 LED 매트릭스의 생성형 패턴을 제어하는 오픈소스 하드웨어
- **컨셉**: 백남준 《참여TV》(1963)의 현대적 재해석, LED 신디사이저
- **현재 상태**: 프로토타입 단계. PCB 주문 들어간 상태, 검증 전. 공개 발사는 2~3주 뒤 예정
- **이 작업 목적**: 공개 발사 전에 저장소 **뼈대만** 세팅. 콘텐츠 채우는 건 이후 작업

## 전체 전략

모노레포 구조. 하드웨어, 펌웨어, 웹이 한 저장소에서 함께 버전 관리됨.

- **Firmware**는 Arduino IDE 기반 (.ino 단일 파일). 당분간 이대로, 나중에 PlatformIO로 전환 가능성 있음.
- **Web**은 Next.js 프로젝트. 현재 랜딩 페이지지만 추후 ESP32 웹 플래셔 + 패턴 공유 플랫폼으로 확장 예정. 그래서 `landing/`이 아니라 `web/` 네이밍.
- **Hardware**는 Blender 3D 원본, KiCad PCB 파일 포함.
- **Origin 브랜치**는 이미 존재하는 Patternflow: Origin 아카이브. 건드리지 않음.

```

│   │   ├── source/             # .blend 원본

```

빈 폴더는 `.gitkeep` 빈 파일을 넣어 추적.

## 작업 단계

### 1. 저장소 생성
- GitHub에 `patternflow` 이름으로 **public** 저장소 생성
- README / .gitignore / License 자동 생성 옵션은 **모두 체크 해제** (직접 만들 것)
- 로컬에 `git clone`

### 2. 폴더 구조 생성
위 트리대로 빈 폴더 생성. 각 빈 폴더에 `.gitkeep` 파일 삽입.

### 3. 루트 파일 생성
아래 "파일 명세" 섹션의 내용대로 각 파일 작성.

### 4. 기존 자산 배치
사용자가 이 작업 시점에 전달할 자산:

| 자산 | 배치 위치 |
|------|---------|
| 케이스 Blender 원본 (.blend) | `hardware/case/source/` |
| 개별 STL 파일들 | `hardware/case/parts/` |
| 통합 STL 2개 (프린팅용) | `hardware/case/print-ready/` |
| KiCad 파일 (회로도, PCB) | `hardware/pcb/` |
| Arduino .ino 파일 (단일 파일) | `firmware/patternflow.ino` |
| 기존 Next.js 웹 프로젝트 (version2) | `web/` 전체 이동 |

**웹 이동 시 주의**: Next.js 프로젝트 루트(package.json, next.config.js 등이 있는 지점)가 `web/` 바로 아래가 되어야 함. Vercel Root Directory 설정과 일치시켜야 빌드됨.

### 5. Git 커밋 및 푸시
```
git add .
git commit -m "Initial repository structure"
git push origin main
```

### 6. 개발 브랜치 생성
```
git checkout -b dev
git push -u origin dev
```

이후 모든 작업은 `dev` 브랜치. `main`은 안정 상태만 유지.

### 7. Vercel 연결
- Vercel에서 `patternflow` 저장소 Import
- **Root Directory**: `web` 으로 지정 (중요 — 모노레포 설정)
- Framework: Next.js (자동 감지될 것)
- `main` 브랜치 → 프로덕션
- `dev` 및 기타 브랜치 → 자동 preview URL 생성
- 도메인 연결은 사용자가 직접

An LED synthesizer. Play light patterns with your fingertips.

## 주의사항

1. **Origin 브랜치 건드리지 말 것.** 별도로 관리 중인 아카이브임.
2. **web/ 이동 시 package.json 경로 주의.** Next.js 프로젝트 루트가 `web/` 바로 아래여야 Vercel이 빌드 가능.
3. **Release 태그는 아직 만들지 말 것.** PCB 검증·펌웨어 확정 후 사용자가 직접.
4. **bom.md 내용은 비워둘 것.** 어필리에이트 링크 채우기는 사용자 작업.
5. **실제 URL 값(도메인, Discord 초대 링크 등)은 README에 placeholder로 두기.** "coming soon"으로 표기.
